# 🚀 AI Job Hunting (AI 工作猎手) 部署指南 v1.0

本项目是一个基于 AI 赋能的求职辅助工具，包含 **Java 后端 (Server)** 和 **浏览器插件 (UserScript UI)** 两部分。本指南将帮助您从零开始完成生产环境部署。

---

## 🛠️ 环境要求

- **操作系统**: 推荐 Linux (Ubuntu/CentOS)，支持 Docker 环境。
- **运行环境**: 
  - **JDK 17+**: 后端基于 Spring Boot 3 开发。
  - **MySQL 5.7+**: 数据持久化。
  - **Node.js 18+ & pnpm**: 用于前端脚本构建。
  - **Docker & Docker Compose**: 容器化一键部署。
- **证书要求**: **必须配置 SSL (HTTPS)**。由于 Boss 直聘网站使用 HTTPS 协议，插件调用的 API 接口也必须通过 HTTPS 访问，否则浏览器将拦截请求(本地部署不需要)。

---

## 📂 1. 数据库初始化

1. 创建数据库 `ai_job`。
2. 运行脚本：执行 `ai-job-hunting-server/src/main/resources/schema.sql` 完成表结构创建。

---

## ⚙️ 2. 后端配置 (application.properties)

在 `ai-job-hunting-server/src/main/resources/application.properties` 中修改以下核心配置。

> **⚠️ 安全警告**: 在部署前，请务必修改默认的 API Key、私钥和密码！(默认都是xxx,可批量搜索)

### A. 数据库连接
```properties
spring.datasource.url=jdbc:mysql://<数据库IP>:3306/ai_job?useUnicode=true&characterEncoding=utf-8&serverTimezone=UTC&useSSL=false
spring.datasource.username=<用户名>
spring.datasource.password=<密码>
```

### B. AI 配置 (必须配置)
1. **Kimi (Moonshot)**: 用于简历 OCR 识别。请在 [Moonshot 平台](https://platform.moonshot.cn/) 申请 API Key。
   ```properties
   spring.ai.kimi.api-key=<YOUR_KIMI_API_KEY>
   ```
2. **OpenAI 模型池**: 用于 AI 聊天的 API Key 池，支持多模型轮询。
   ```properties
   openai.pool.config.list = [
     {
       "base-url": "https://api.openai.com/v1",
       "api-key": "sk-xxx",
       "name": "gpt-4o",
       "completions-path": "/chat/completions",
       "chat.options.model": "gpt-4o"
     }
   ]
   ```

### C. 支付宝支付 (可选)
如果需要开启收费模式（赚米），需配置支付宝参数：
```properties
alipay.app_id=<支付宝APPID>
alipay.merchant_private_key=<商户私钥>
alipay.alipay_public_key=<支付宝公钥>
alipay.notify_url=https://<你的域名>/api/pay/paySuccessCallBack
```

### D. 配置中心登录 (SmartConfig)
动态参数管理后台的登录凭证：
```properties
smart.username=admin
smart.password=<修改为你自己的密码>
```

## 🔧 3. 动态业务配置 (SmartConfig UI)

项目部署成功后，可以通过以下方式动态管理业务参数，无需重启：

- **访问地址**: `http://<服务器IP>:6768`
- **主要配置项**:
  - **产品配置 (`generate.order.group.config`)**: 定义支付套餐、点数等。
  - **AI 策略 (`openai.pool.config.list`)**: 配置坐席自动回复、自动投递过滤等场景使用的模型。

---

## 🐳 4. Server 部署 (Docker)

- 可直接通过jar包部署
- 也可通过docker-compose部署仔细打包

*默认端口*: API (9100), Config UI (6768)。

---

## 🧩 5. 插件安装 (UI)

### 方案 A：直接安装
将根目录下的 `ai-job-hunting.user.js` 内容复制，并在浏览器的 **Tampermonkey** 插件中“添加新脚本”。

### 方案 B：自行构建 (推荐)
如果您修改了后端接口地址或 UI 代码：
1. 进入 `ai-job-hunting-ui`。
2. 安装依赖：`pnpm install`。
3. 构建脚本：`pnpm run build`。
4. 在 `dist` 目录下找到生成的 `.user.js` 文件进行安装。
